前言

如果說要開發小型專案的話我個人首推Jupyter Lab(前身為Jupyter Notebook)，相較於vscode、spyter、pycharm，它的優勢在於能即時執行程式碼，而不需要再去cmd下python app.py等指令，再加上它的output效果非常優，無論是pandas、matplotlib、plotly呈現的表格及圖片使我們在數據分析上有超棒的觀賞性。

.

說到安裝套件就必須提一下當初我的痛苦，當時我剛學Python想安裝套件時我呆呆地一個一個pip install，所以你可以想像如果我要安裝20個packages時那畫面有多美了XD
所幸本人的懶人病促使我去搜尋如何一鍵安裝所有想要的套件，所有安裝小訣竅都寫在這篇教學文中囉!!

目錄

《一》安裝miniconda

《二》建立開發環境

《三》安裝套件

《四》下載中文字體

《一》安裝miniconda

1.選擇對應版本安裝
2.開啟anaconda prompt 輸入 conda install jupyterlab
3.設定Jupyter Lab
  • 開啟Jupyter Lab: 在anaconda prompt 輸入jupyter lab
  • 點選Settings ➨ Advanced Settings Editor ➨ 複製System Defaults 貼至 User Preferences
  • 修改lineNumbers:false ➨ lineNumbers:true (寫程式時會顯示行數)

《二》建立開發環境

1.開啟anaconda prompt並輸入 (myenv可自由命名)
conda create -n myenv python=3.7

activate myenv

pip install ipykernel

ipython kernel install --user --name=myenv
2.顯示已建置環境
conda info --env

《三》安裝套件

1.顯示已安裝套件
# 直接顯示
pip list

# 另存文字檔
pip freeze > requirements.txt

2.修改requirements.txt
requests==2.22.0
beautifulsoup4==4.9.3
selenium==3.141.0
tqdm==4.31.1
ipywidgets==7.4.2
seaborn==0.11.1
plotly==5.1.0
pyecharts==1.7.1
tensorflow==1.14.0
scikit-learn==0.24.2
lightgbm==3.1.1
xgboost==1.4.2
keras==2.2.4
yfinance==0.1.62
fugle_realtime==0.2.9
3.安裝全部套件
pip install -r requirements.txt
4.手動安裝
  • pandas & numpy
pip install pandas==1.3.1
pip install numpy==1.21.1
  • talib(下載檔案並儲存至user)
pip install TA_Lib-0.4.20-cp37-cp37m-win_amd64.whl
  • jupyterlab plotly (base環境執行)
jupyter labextension install jupyterlab-plotly@4.14.3

《四》下載中文字體 (TaipeiSansTCBeta-Regular.ttf)

1.搜尋matplotlib字體檔案
import matplotlib

matplotlib.__file__

C:\Users\apple\Anaconda3\envs\tw-tradingview\lib\site-packages\matplotlib\init.py

2.字體檔案(.ttf)儲存至matplotlib/mpl-data/fonts/ttf
3.刪除user/apple/.matplotlib/ 的 fontlist.json
4.使用前輸入
import matplotlib.pyplot as plt

plt.rcParams['font.sans-serif'] = ['Taipei Sans TC Beta']

結論

可能大家對於建置環境會有一些疑問，為什麼不直接安裝在一般環境就好了，還要另外再創建一個新的環境呢?
其實在做每一個專案時都會建議使用獨立的開發環境，原因很簡單，專案A的套件可能會造成專案B的套件不相容，這時系統就有可能會自動幫你更新專案B的套件，而未來在開發專案B時就會發現因為套件版本錯誤造成程式無法執行。
另外，將每一個專案都有自己獨立的環境也會讓使用者方便了解每個專案真正需要的套件是哪一些，而不用下載過多用不到的套件。

.
