前言

如果你學習Python有一段時間後應該會覺得某些open source的套件功能實在不太順手，於是開始改寫套件，開發完畢後發現:哇!自己根本是天才(大多數時間都會覺得自己根本是廢物啊)!
於是將自己的改寫好的專案上傳到GitHub上，但如果想在其他電腦或colab上安裝自己的套件時要輸入:
pip install git+https://github.com/User_Name/User_Repo.git

.

咦!前面課程安裝套件的指令好像沒這麼長欸，以安裝pandas為例，安裝pandas套件是輸入:
pip install pandas
大家有看出差異嗎?安裝pandas只要install套件名稱就好了，那為什麼安裝自己的套件要寫專案的網址呢?
原因在於我們沒有將專案上傳到PYPI(Python第三方軟體包的軟體存儲庫)上，所以無法使用
pip install MyProject

.

而本堂課的重點在於將自己的專案上傳到PYPI上以方便日後下載時的指令快速簡單，如果你對於安裝自己套件的指令沒那麼care的話可以跳過這堂課程哦!

《一》專案架構

tw_tradingview --- tw_tradingview --------------------- __init__.py #有__init__.py的套件才會被打包上傳
                               |                                                                |
                               |                                                                |- main.py #主套件
                               |
                               |- requirements.txt #專案所需套件
                               |
                               |- .gitignore #省略不上傳的資料
                               |
                               |- setup.py #上傳到PYPI的指令
                               |
                               |- README.md

直接使用.gitignore模板

結論

從PYPI上安裝自己套件的感覺是不是很有成就感啊!而且整個上傳過程也比想像中來的容易許多，每次更新上傳只需要短短三行程式碼就解決了!

.

而最後要注意的是更新版本後上傳到PYPI外，還要記得要將最新版本上傳到GitHub上，因為使用pip install時會依照不同的安裝路徑安裝到不同的版本，所以一定要記得更新所有開源的版本哦!
